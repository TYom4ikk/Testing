Создать библиотеку классов, содержащую:

Класс «Комната»(идентификатор комнаты, длина комнаты, ширина комнаты, высота потолков, количество окон, высота окна, ширина окна).

Класс «Обои»(идентификатор обоев, длина, ширина, цена)

Методы класса:

«Информация о комнате»– выводит параметры заданной комнаты на экран;
«Ремонт»- определяет количество рулонов обоев, необходимых для ремонта комнаты с заданными параметрами. Предусмотреть работу с рулонами разной длины/ширины (Например, 10м*1,5м и 15м*1м). Предусмотреть тот факт, что количество обоев для комнаты рассчитывается от площади комнаты, длины рулона и вычитает размеры окон)
Предусмотреть тот факт, что количество обоев для комнаты рассчитывается от площади комнаты, длины рулона и вычитает размеры окон)

Выполните расчёт площади стен комнаты по формуле:
2 * (ШИРИНА КОМНАТЫ + ВЫСОТА КОМНАТЫ) * ВЫСОТА КОМНАТЫ)

Выполните расчёт площади окон по формуле:
КОЛИЧЕСТВО ОКОН * ( ШИРИНА ОКНА* ВЫСОТА ОКНА)

Выполните расчёт площади комнаты, которую необходимо оклеить обоями (вычисление эффективной площади стен) по формуле:
ПЛОЩАДЬ СТЕН - ПЛОЩАДЬ ОКОН

Выполните расчёт необходимого количество рулонов обоев по формуле:
ОКЛЕИВАЕМАЯ ПЛОЩАДЬ / ПЛОЩАДЬ ОДНОГО РУЛОНА.

(Будем предполагать, что площадь одного рулона – 5 ). Если получилось дробное число, то необходимо округлить до целого числа в большую сторону (например, если расчет показал, что необходимо 5.3 рулона, то вы должны приобрести 6 рулонов)

«Оптимальный размер рулона» - при вводе массива
с информацией об обоях, определяет по заданным
параметрам комнаты оптимальный тип обоев

 

Для тестирования работы класса «Комната» выполните создание необходимых методов и выполните следующие тестовые задачи:

1.       Тест на проверку правильности подсчета количества рулонов
для комнаты с одним окном и заданной длиной обоев (например, 10м):
Ожидаемый результат: метод «Ремонт «корректно отражает количество
необходимых рулонов.

2.       Тест на проверку правильности подсчета количества
рулонов для комнаты без окон и заданной длиной обоев (например, 15м):
Ожидаемый результат: метод «Ремонт «корректно отражает
количество необходимых рулонов.

3.       Тест на проверку правильности подсчета количества
рулонов с нулевой высотой окна:
Создание комнаты с окном высотой 0.
Ожидаемый результат: метод «Информация о комнате» корректно
отражает высоту окна как 0 и не выдает ошибок.

4.       Тест на проверку ввода нечисловых значений
параметров комнаты:
Ожидаемый результат: класс должен предотвращать создание
комнаты с отрицательными значениями и вернуть соответствующее
сообщение об ошибке.

5.       Тест на проверку правильности подсчета количества
рулонов при попытке создания комнаты с отрицательными
или нулевыми значениями длины, ширины или высоты.
Ожидаемый результат: класс должен предотвращать создание
комнаты с отрицательными значениями и вернуть соответствующее
сообщение об ошибке.

6.       Тест добавление в коллекцию объектов класса
«Комната» (возвращает коллекцию);
Создать массив объектов класса «Комната «с различными
значениями атрибутов;
Проверить, что массив был успешно создан и содержит
заданное количество элементов.

7.       Тест на поиск комнат по идентификатору
Выполнить поиск комнат по идентификатору. Вызвать метод
«Информация о комнате»;
Проверить, что результат поиска вернет только одну комнату.

 8.       Тест на обновление информации о комнате:
Создать объект класса «Комната «.
Обновить информацию о комнате с помощью метода установки
значений атрибутов.
Проверить, что информация была успешно обновлена. 

9.       Тест на удаление информации о комнате по идентификатору:
Создать объект класса «Комната».
Вызвать метод удаления по идентификатору;
Проверить, что информация была успешно удалена.

10.   Тест на подсчет «Оптимальных размеров рулона»:
Вызвать метод «Оптимальный размер рулона»;
Проверить, что он успешно выполнен.
